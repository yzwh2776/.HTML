Introduction to Network Devices I:

The Open Systems Interconnection (OSI)

  • Layer 7 - Application
  • Layer 6 - Presentation
  • Layer 5 - Session
  • Layer 4 - Transport
  • Layer 3 - Network
  • Layer 2 - Data Link
  • Layer 1 - Physical

Most devices function at more than one level of the OSI model, but when determining where they fit into the model, we look at the highest level.

Layer 1 Devices

  • Analog Modem

    • Modems take a digital signal from a digital node and convert it to an analog signal (modulating the signal). In return it takes an analog signal and converts to a digital signal (demodulating the signal).

    They were developed to create connection vie that Public Switched Telephone Network (PSTN) using the Plain Old Telephone system (POTS).

    Modems provide for a single connection to a network and are only concerned with the wire, and the wire resides on Layer 1 of the OSI model. The modem doesn't care where the signal comes from, it just does it's job.

  • Hub

    • A hub function as a concentrator/repeater in that it doesn't care where the signal comes from or where it's going. It takes a signal arriving on a port and replicates that signal out all of the other ports.

Layer 1 devides communciate with wires.

Layer 2 Devices

  • Switch

    • A switch utilizes an application-specific integrated circuit (ASIC) chip. The ASIC chip has specific programing that allows the switch to learn when a device is on the network and which ports itis connected to via that devices's layer 2 MAC address.

  • Wireless Access Point (WAP)

    • A WAP is a specific type of network bridge that connects (bridges) wireless network segments with wired network segments.

Layer 2 devices only communicate with local network devices

Layer 3 devices

  • Multilayer Switch (MLS)

    • An MLS provides normal layer 2 network switching services, but it will also provide layer 3 or higher OSI model services. The most common MLS is a layer 3 switch, which not only utilizes an ASIC chip for switching, but that ASIC chip is also programed to handle routing functions. This allows the device to communicate and pass data to non-local network devices.

  • Router

    • A router is the most common network device for connecting different networks together utilizing the OSI model's layer 3 logical network information. The router uses software programmin for decision making as compared to the switch's use of an ASIC chip. The router uses this programming to keep track of different networks and what it considers to be the best possible route to reach those networks.

Layer 3 devices can communcate with both local and non-local network devices.

Introduction to Network Devices II

Security Devices

  • Firewall

    • A firewall can be placed on routers or hosts (software based) or can be its own device and it functions at multiple layers of the OSI model (2,3,4,7). The firewall blocks packets from entering or leaving the network via stateless inspection or via stateful inspection.

    • Stateless Inspection

      • The firewall will examine every packet against a set of rules. Once the packet matches a rule, the rule is enforced, and the specific action is taken.

    • Stateful Inspection

      • The firewall will only examine the state of the connection between networks. Specifically, when a connection is made from an internal network to an external network, the firewall will not examine any packets returning from the external connection. As a general rule, external connections are not allowed to be initiated with the internal network.

    The firewall is the first line of defense in protecting the internal network from outside threats.

  • Intrusion Detection System (IDS)

    • An IDS is a passive system designed to identify when a network breach or attack agains the network is occurring. This system is usually designed to inform a network administrator when a breach or attack has occurred through log files, SMS, and/or email. An IDS cannot prevent or stop a breach or attack on its own. The IDS receives a copy of all traffic and evaluates it against a set of standards. The standards it uses may be, Signature based, Anomaly based, Policy based.

    • Signature Based

      • Evaluates network traffic for known malware or against attack signatures.

    • Anomaly Based

      • Evaluates network traffic for suspicious changes.

    • Policy Based

      • Evaluates network traffic against a specific declared security policy.

    The IDS may be depolyed at the host level.

  • Intrusion Prevention System (IPS)

    • An IPS is an active system designed to stop a breach or attack from succeeding in damaging the network. They are usually designed to perform an action or set of actions to stop the malicious activity. Like the IDS they will inform a network administrator through the use of log files, SMS, and/or email. All traffic on the network segment flows through the IPS to either enter or leave the segment, and like the IDS, all traffic is evaluated against a set of standards. The best placement for an IPS on a network is between a router (with a firewall) and the destination network segment.

    Some of the ways an IPS is programmed to make an active response to the situation are:

    • Block the offending IP address
    • Close down the vulnerable interface
    • Terminate the network session
    • Redirect the attack

    IPS are designed to be active and to stop the breach or attack.

  • Virtual Private Network (VPN)

    • A VPN concentrator will allow for many more secure VPN connections to a network. The concentrator will provide proper tunneling and encryption, depending on the type of VPN connection that is allowed. Outside of internet transactions (which use SSL VPN connections at layer 7) most concentrators will function at the network layer (layer 3) of the OSI model, providing IPsec encryption through a secure tunnel.

Optimization and Performance Devices

  • Load Balancer

    • A load balancer may also be called a content switch or content filter. A load balancer is a network application that is used to load balance between multiple hosts that contain the same data - spreading out the workload for greater efficiency. They are commonly used to distribute the requests (workload) to a server farm among the various servers, helping to ensure that no single server gets overloaded.

  • Proxy Server

    • A proxy server is an application that requests resources on behalf of client machines. It is often used to retrieve resources from outside untrusted networks on behalf of the requesting client. It hides and protets the requesting client, can be utilized to filter allowed content, and can increase network performance by caching commonly requested web pages.
